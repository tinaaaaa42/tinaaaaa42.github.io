优先考虑基于任务而非基于线程

由此可见基于线程是无法获取返回值的，而基于任务可以使用返回的std::future的get函数获取返回值，从而可以处理各种异常。
而基于线程的实现在出现异常时会直接调用std::terminate，即使声明了noexcept也不例外。

基于任务的实现实际上是一种封装，它将线程的调度交给了库的开发者实现。

使std::thread在所有路径最后都不可结合

可结合(joinable)状态的std::thread对应于正在运行或者可能要运行的异步执行线程。相反的，不可结合的std::thread对象包括：

  • 默认构造的std::thread对象
  • 已经被移动走的std::thread对象
  • 已经被join或detach的std::thread对象
